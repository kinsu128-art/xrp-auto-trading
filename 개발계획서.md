## 1. 개요

### 1.1 프로젝트 목표
- 래리 윌리엄스 돌파 전략을 활용한 XRP 자동매매 시스템 개발
- 빗썸 API와 텔레그램 봇을 통한 실시간 매매 및 알림 구현
- 백테스트를 통한 전략 검증 및 최적화

### 1.2 개발 기간
- **총 기간**: 약 6주
- **실전 테스트 포함**: 약 7주

### 1.3 핵심 성과 지표 (KPI)
- 백테스트 승률: 60% 이상
- 백테스트 연간 수익률: 20% 이상
- 시스템 가용성: 99.9% 이상
- 매매 실행 지연: 1분 이내

## 5. 테스트 전략

### 5.1 단위 테스트 (Unit Tests)
- **범위**: 각 모듈별 독립 기능
- **도구**: pytest
- **커버리지 목표**: 80% 이상
- **실행 주기**: 개발 시 매번

```python
# tests/test_strategy_engine.py
def test_check_buy_condition_all_true():
    strategy = LarryWilliamsStrategy()
    candles = create_mock_candles(meet_all_conditions=True)
    result = strategy.check_buy_condition(candles)
    assert result['should_buy'] == True
    assert result['reasons'] == []

def test_check_buy_condition_condition1_false():
    strategy = LarryWilliamsStrategy()
    candles = create_mock_candles(condition1=False)
    result = strategy.check_buy_condition(candles)
    assert result['should_buy'] == False
    assert 'breakthrough_not_met' in result['reasons']
```

### 5.2 통합 테스트 (Integration Tests)
- **범위**: 모듈 간 연동
- **테스트 케이스**:
  1. 캔들 마감 → 매수 신호 → 매수 실행 → 알림
  2. 매수 → 캔들 마감 → 매도 실행 → 알림
  3. API 장애 시 복구
  4. 네트워크 단절 시 재시도

```python
# tests/test_integration.py
def test_buy_flow():
    bot = TradingBot(config)
    bot.on_candle_close(mock_candles_with_buy_signal)
    assert mock_executor.market_buy_called
    assert mock_notifier.send_buy_signal_called
```

### 5.3 백테스트 (Backtesting)
- **범위**: 전략 검증
- **데이터**: 최소 1년치
- **지표**:
  - 수익률, 승률, 최대 손실률
  - 샤프 비율, 손익비
  - 총 거래 횟수

```python
# tests/test_backtest.py
def test_backtest_performance():
    candles = load_historical_data(days=365)
    backtester = BacktestRunner(strategy, candles)
    result = backtester.run()
    assert result['win_rate'] >= 0.60
    assert result['max_drawdown'] <= 0.10
```

### 5.4 실전 테스트 (Live Testing)
- **범위**: 실전 환경 검증
- **기간**: 1주일
- **금액**: 소액 (10,000 ~ 50,000 KRW)
- **모니터링 항목**:
  - 시스템 안정성
  - 매매 실행 지연
  - 알림 도착 시간
  - 에러 발생률
